# 📱 手机端打包 APK 教程（GitHub Actions）

没有电脑也能打包！用手机操作 GitHub，让服务器自动帮你构建 APK。

---

## 📋 准备工作

1. 一个 **GitHub 账号**（没有的话先去 github.com 注册）
2. 手机浏览器（推荐 Chrome 或 Safari）

---

## 🚀 详细步骤

### 第一步：上传代码到 GitHub

#### 方法 A：用 GitHub 网页端手动上传（推荐，最省事）

1. 打开手机浏览器，访问 https://github.com/new
2. 登录你的 GitHub 账号
3. 填写仓库信息：
   - **Repository name**：填 `mht-to-pdf`（随便起个名字）
   - **Public** / **Private**：选 Public 就行
   - 勾选 **Add a README file**
4. 点击 **Create repository** 创建仓库

5. 进入刚创建的仓库页面，点击 **Add file** → **Upload files**
6. 打开文件管理器，找到 `mht_to_pdf_app` 文件夹
7. 把里面**所有文件和文件夹**都选中，拖到上传区域（或者一个个选）
   
   ⚠️ 注意：要包含这些文件和文件夹：
   - `.github/` 文件夹（里面有 workflows）
   - `lib/` 文件夹
   - `android/` 文件夹
   - `pubspec.yaml`
   - `analysis_options.yaml`
   - `.gitignore`

8. 拉到页面底部，点击 **Commit changes** 上传

#### 方法 B：用手机 Git 应用（比较麻烦）

如果手机上有 Termux 或其他 Git 工具，可以 clone 后 push。

---

### 第二步：自动构建 APK

代码上传后，GitHub 会自动开始构建：

1. 进入你的仓库页面
2. 点击顶部的 **Actions** 标签页
3. 你会看到一个正在运行的工作流，名字叫 **Build APK**
4. 等待它跑完（大约 5-10 分钟）
   - 黄色圆点 = 正在构建
   - 绿色对勾 = 构建成功
   - 红色叉号 = 构建失败

---

### 第三步：下载 APK

构建成功后：

1. 点击那个绿色对勾的工作流
2. 拉到页面最底部，找到 **Artifacts** 区域
3. 点击 **mht-to-pdf-apk** 下载
4. 下载的是一个 zip 包，解压后就是 `app-release.apk`

---

### 第四步：安装 APK

1. 找到下载的 `app-release.apk` 文件
2. 点击安装
3. 如果提示「未知来源」，允许安装即可
4. 安装完成后打开使用

---

## ❓ 常见问题

### Q: 构建失败了怎么办？
A: 点击失败的工作流，查看日志里的错误信息，把错误截图或者复制出来反馈。

### Q: 上传文件太麻烦，有没有更简单的方法？
A: 可以在手机上装个「GitHub」APP，操作会方便一点。或者用 Termux 命令行操作。

### Q: 下载的 zip 包怎么解压？
A: 手机一般自带解压功能，找到文件点击就能解压。或者用「ZArchiver」等解压 APP。

### Q: 安装时提示解析包错误？
A: 可能是下载不完整，重新下载试试。或者检查手机 Android 版本（需要 5.0 以上）。

### Q: 能改应用名字和图标吗？
A: 可以。改名字修改 `android/app/src/main/AndroidManifest.xml` 里的 `android:label`。图标比较复杂，建议先用默认的。

---

## 🔧 如果需要修改应用

1. 在 GitHub 仓库里找到要改的文件
2. 点击右上角的铅笔图标编辑
3. 修改后拉到底部点击 **Commit changes**
4. Actions 会自动重新构建，等几分钟就能下载新的 APK 了

---

## 💡 小提示

- 第一次构建可能慢一点，后面有缓存会快很多
- 构建过程中不用一直等，关掉页面也没事，GitHub 会在后台跑
- 免费版 GitHub Actions 每月有 2000 分钟的额度，完全够用
